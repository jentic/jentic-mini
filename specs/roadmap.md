## Phase 1 — Local Service Routing

**Goal:** Allow the broker to route to non-public hostnames (`localhost`, bare names, IP addresses).
**Depends on:** none (self-contained broker change)
**Priority:** High

Today the broker identifies upstream targets by requiring the first path segment to contain a
dot. `localhost`, `my-service`, or `192.168.1.10` are rejected or treated as self-referential —
so agents cannot use Jentic Mini to call home-automation APIs, internal tools behind a firewall,
local dev servers, or any self-hosted service without a public domain. OpenAPI 3.1 Server
Variables provide the native mechanism for this.

- Add optional `base_url` override to `POST /import`, stored in `apis.base_url` (column already exists)
- Honour OpenAPI 3.1 `servers[0].url` with `{serverUrl}` variables at import time
- Assign a stable broker alias (e.g. `my-service.local`) derived from the import label
- Modify the broker to resolve the target URL from `apis.base_url` at request time rather than trusting the path segment literally
- Update the dot-based host detection to allow configured aliases to bypass it
- Add integration tests: import a spec with `base_url`, broker a request, verify it routes to the overridden URL
