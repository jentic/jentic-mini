# SDD Lessons

Recurring lessons learned from implementation that future spec authors should know about. Loaded by `/sdd-new-spec` and `/sdd-new-phase` before scaffolding so new specs absorb the learning.

This file is **not part of the constitution** (the constitution is mission + tech-stack + roadmap — see `.claude/rules/sdd-constitution.md`). It is an operational, append-only checklist that grows as the team ships phases.

## Lifecycle

- **Captured** in `specs/<date>-<slug>/retrospective.md` after an implementation PR surfaces a spec gap. The retrospective is human-written; `/sdd-implement-spec` only emits raw material in its PR body's `## Spec deviations` section.
- **Distilled** into this file by `/sdd-distill-lessons` (manual cadence): the skill reads all retrospectives, deduplicates, proposes additions via `AskUserQuestion`, writes on confirmation.
- **Graduated** to `specs/tech-stack.md` (manual edit) when a lesson recurs across multiple specs and names a load-bearing invariant rather than an empirical reminder. `/sdd-distill-lessons` flags promotion candidates; the promotion itself is a hand-rolled edit.
